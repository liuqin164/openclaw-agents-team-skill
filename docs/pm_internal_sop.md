# PM 内部判断 SOP
PM Internal Decision SOP

本文件定义 PM 在会话中判断：
- 是否恢复上次开发
- 是否触发方向性变更
- 是否开启新主线

---

## 一、会话状态定义

PM 必须在内部维护以下三种状态之一：

- active_dev：正在进行开发
- paused_dev：开发已暂停
- closed_dev：开发已结束

---

## 二、开发锚点（Development Anchor）

开发进度始终绑定到最近的稳定 Artifact：

- PRD（approved / draft）
- ChangeRequest（draft / approved）

这是恢复开发的唯一依据。

---

## 三、“恢复 or 新方向”5 问判断表

### Q1：用户是否明确提到“继续之前的项目 / PRD”？
- 是 → 倾向恢复

### Q2：新描述是否仍基于原目标用户？
- 否 → 方向性变更信号

### Q3：新需求是否仍服务于原 PRD 的核心目标？
- 否 → 新方向

### Q4：如果接受新需求，原 PRD 是否仍有 50% 以上内容有效？
- 否 → 新主线

### Q5：用户是否关心“之前做的东西还算不算”？
- 是 → 优先恢复或继承

---

## 四、快速决策规则

- ≥2 个核心否定（Q2 / Q3 / Q4） → 新 PRD 主版本
- 用户明确说“继续” → 无条件恢复
- PM 不确定 → 生成 ChangeRequest 请求确认
